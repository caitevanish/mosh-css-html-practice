2- CSS Best Practices

Best Practices:

  1. Follow a naming convention consistently
    • Kebab case: .nav-bar
    • Camel case: .navBar
    • Pascal case: .NavBar
    • Snake case: .nav_bar
  2. Create logical sections in your stylesheet
    • Separate style sheets in different sections
    • Add comments in .css to group ie. typography, body, etc.
  3. Avoid over-specific selectors
    • Common problem
    • Remove direct child selectors, make selectors more relaxed.
    • Remove type of HTML elements
    • Rename li and give class name as 'item' remove redundancy
    • !important could leak into other similar elements that already have their own styling.
      • specify the element that will be changed by using a prefix to specify the element within a component .nav .nav-item{ }
      • rename the class with the new prefix
  4. Avoid !important keyword.
    • chances are you're getting too specific
  5. Sort CSS properties
    • Unsorted properties are hard to read
    1. Select properties
    2. Command palette > 'sort' > 'Sort lines ascending' (this puts in alphabetical order)
  6. Take advantage of style inheritance
    • Apply the style to parent elements
  7. Extract repetitive patterns
    • Object Oriented CSS
  8. Avoid repetive values in code
    • DRY: Don't repeat yourself

3- Variables

  • Use CSS variables to get rid of duplications
  • Use the root pseudo class selector matches HTML element. Define custom properties or global variables
  • Start with 2 hyphens and then give variable custom property or name

    • :root{
    --color-primary: #fff;
    --border-size: 2px;
    }

    .one{
    background: #kkk
    }
  • These custom properties will be accessed by all elements in the DOM.
  • Access the bavkground using the var function.

    • .one{
    background: var(--color-primary)
    }

4- Object-oriented CSS

  • Reusable components.
  • Structure styles to make them reusable.
  • End up wiht a smaller, more maintable style sheets

Principles:

  1. Separate container from content

    2. .hero .btn {
    background: gold;

    border: 0;
    border-radius: 5px;
    padding: 5;
    font-family: 'Gill Sans', 'Gill Sans MT', Calibri, 'Trebuchet MS', sans-serif;
    }
    • Sidebar = aside HTML element
    • btn = content, hero = container
    • Separate content from the container!
    • remove .hero from rule and it will apply to all btns (ie. in hero AND in aside/sidebar)
  2. Separate structure and skin
    • Structure = ie. border, border-radius, padding, font-family, etc.
    • Skin = ie. color
    • Use abstract names for classes (ie. btn-primary > btn-gold)

HTML:

< header class="hero">
< button class = "btn btn-primary">Sign-Up< /button>

< aside class="side-bar">
< button class="btn btn-secondary">Contact
< /header>

CSS:

.btn {
border: 0;
border-radius: 5px;
padding: 5;
font-family: 'Gill Sans', 'Gill Sans MT', Calibri, 'Trebuchet MS', sans-serif;
}

.btn-primary{
background: gold;
}

.btn-secondary{
background: blue;
}

5- BEM

  • BEM = Block element modifier
  • Elements that can be modified

Card

Elements only make sense in the context of their block, not supported outside of block.

< div class="card card--popular">
< header class="card__header">
< span class="card__price">
< div class="card__body">
< button class="btn">
< /div>
< /span>
< /header>
< /div>

Modify

  • Class gets double hyphen and modified name
  • Use double hyphens to separate block from a modifier and 2 underscores to separate block from an element
